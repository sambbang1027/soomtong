## 디자인 원칙

1. **단일 화면** — 스크롤 없이 승강장에서 한 손으로 한 눈에 파악
2. **모바일 퍼스트** — 375px~430px 기준 설계, 태블릿·데스크탑은 중앙 정렬로 대응
3. **1초 인지** — 색상만으로 혼잡도를 즉시 파악 가능해야 함

## 컴포넌트 상세

### StationHeader
- 좌측: 로고 `🌬 숨통` (고정)
- 우측: 역명 + 호선 뱃지 (호선 컬러 배경) + 드롭다운 화살표
- GPS 감지 중: "위치 확인 중..." 스피너
- 탭 시 역 검색 모달 오픈

### RecommendBanner
- 카드 형태, 배경은 추천 칸 혼잡도 색상을 10% 불투명도로 tint
- 상단: 아이콘 + "지금은 N번 칸이 가장 숨통 트입니다!"
- 하단: 혼잡도 % + 단계 레이블 (여유 / 보통 / 혼잡)
- 공동 최저 혼잡 칸이 여러 개면 "1번·9번 칸" 형태로 병기

### SubwayGrid
- 가로 스크롤 가능 (모바일에서 10칸이 잘릴 때 대응)
- 열차 진행 방향 표시 (← 화살표 텍스트)
- 추천 칸은 `scale-105` + 흰 테두리 + `BEST` 뱃지로 강조

### CarBlock
- 정사각형 비율, 둥근 모서리 (`rounded-lg`)
- 배경색: 혼잡도 단계에 따라 emerald / amber / red
- 내부: 혼잡도 % (큰 글씨, 흰색)
- 하단: 칸 번호 (보조 텍스트)
- 색상 전환 시 `transition-colors duration-500` 적용

## 반응형 대응

| 화면 너비 | 레이아웃 |
|----------|---------|
| ~430px (모바일) | 풀 너비, 패딩 `px-4` |
| 431px~768px (태블릿) | `max-w-sm` 중앙 정렬 |
| 769px~ (데스크탑) | `max-w-sm` 중앙 정렬, 나머지 영역 배경만 표시 |

SubwayGrid는 10칸이 화면 너비를 넘을 경우 가로 스크롤(`overflow-x-auto`) 처리.
